# D&D RAG Workflow

## Purpose

Use this workflow for chat, retrieval, markdown rendering, and future D&D card or action features.

## Rules

- Keep responses markdown-ready
- Keep card rendering extensible
- Keep action controls separate from message text
- Preserve source metadata for future citations

## Good Fits

- Chat message rendering
- Spell, monster, weapon, item, character, and NPC cards
- Action buttons like roll, cast, inspect, or apply effect
